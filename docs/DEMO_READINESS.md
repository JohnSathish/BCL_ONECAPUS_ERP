# Live Demo Readiness Guide

Use this checklist before showing BCL OneCampus to the Principal, staff, or on a live server.

---

## One command to prepare everything

From the repo root (requires Docker Postgres + Redis running):

```bash
npm run demo:prepare
```

This runs:

1. Shift II timetable finalize + publish (11 slots, faculty, rooms)
2. Sem 3/5 student registrations (rosters for attendance)
3. Student academic standings (for fee engine)
4. Published demo exam results (student `/student/results`)
5. Monthly fee demand generation
6. LMS workspace provisioning
7. Smoke tests: attendance, student portal, NAAC, governance

**Quick mode** (skip verify scripts):

```bash
npm run demo:prepare:quick
```

## Pre-demo checklist (15 minutes)

- [ ] `docker compose up -d postgres redis`
- [ ] `npm run dev` (or production stack)
- [ ] `npm run demo:prepare` — all smoke tests pass
- [ ] Browser: use Incognito or disable Grammarly (avoids hydration warning)
- [ ] Rehearse [DEMO_SCRIPT.md](./DEMO_SCRIPT.md) Sections 1–12 only
- [ ] Do **not** click: Backup Restore, Report Builder, Staff Communication placeholders

## Troubleshooting

| Symptom                              | Fix                                     |
| ------------------------------------ | --------------------------------------- |
| Empty attendance roster              | `npm run demo:prepare`                  |
| Student fees blank                   | Re-run `demo:prepare` (monthly demands) |
| Student results empty                | Re-run `demo:prepare` (exam seed)       |
| Faculty timetable shows course codes | Restart API after latest deploy         |
| Governance 400 on members            | Apply migrations + re-seed              |
| Port in use                          | `npm run dev:clean`                     |

## Live server deploy

1. Pull latest code + run migrations
2. Set production `.env` (see [DEPLOYMENT.md](./DEPLOYMENT.md))
3. `npm run demo:prepare` on server (or `db:seed` on fresh DB)
4. Smoke test login at `/admin` and `/staff/academic/timetable`
